API séjours

Récupération de la liste des séjours (toutes identités confondues)

GET api/sejours.json?token=#{token}

Si le paramètre token est spécifié, seuls les séjours ajoutés ou modifiés depuis ce token seront retournées.

L'API retourne un code 200 ("OK"), la liste des séjours triés par date de création (les plus récents en premiers) et le dernier token utilisé.

{
  token: 2587,
  data: [{
    [{...}],
    [{...}],
    ...
  }]
}

Récupération de la liste des séjours (d'une identité)

GET api/identites.json/#{ipp}/sejours.json

Le paramètre obligatoire ipp permet de cibler l'identité dont l'on souhaite obtenir les séjours.

L'API retourne un code 200 ("OK") et la liste des séjours triés par date de création (les plus récents en premiers).

Récupération d'un séjour

GET api/sejours.json/#{iep}

Le paramètre obligatoire iep permet de cibler le séjour à retourner.

L'API retourne un code 200 ("OK") et le séjour demandé.

Ajout d'un séjour

POST api/sejours.json

Une structure séjour doit être passée en paramétre.

L'API retourne un code 201 ("Created").

Modification d'un séjour

PUT api/sejours.json/#{iep}

Le paramètre obligatoire iep permet de cibler le séjour à modifier.
Une structure séjour doit être passée en paramétre.

L'API retourne un code 204 ("No Content").

Suppression d'un séjour

DELETE api/sejours.json/#{iep}

Le paramètre obligatoire iep permet de cibler le séjour à supprimer.

L'API retourne un code 204 ("No Content").
